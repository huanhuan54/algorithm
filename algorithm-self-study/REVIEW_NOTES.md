# 错题与复盘记录

## 二分查找边界

问题：

二分查找容易写出死循环或漏掉最后一个元素。

修正：

本仓库统一使用闭区间 `[left, right]`。只要 `left <= right`，说明区间里仍然有候选元素。每次排除中点后，必须写成 `left = mid + 1` 或 `right = mid - 1`。

补充：

查找插入位置时改用左闭右开区间 `[left, right)` 更容易写边界。`lower_bound` 返回第一个大于等于目标值的位置，`upper_bound` 返回第一个大于目标值的位置。

## 滑动窗口更新顺序

问题：

固定窗口右移时，如果只加新元素、不减旧元素，窗口和会越来越大。

修正：

窗口从 `[right - window_size, right - 1]` 移到 `[right - window_size + 1, right]`，所以更新公式是：

```python
current += nums[right] - nums[right - window_size]
```

## 括号匹配的最终检查

问题：

只在遇到右括号时检查匹配，可能会把 `"((("` 误判为合法。

修正：

遍历结束后必须返回 `not stack`，确认所有左括号都已经被匹配。

## 哈希表两数之和

问题：

如果先把所有数放进字典，遇到目标值刚好是两倍当前值时，可能误用同一个元素。

修正：

边遍历边查询。先查 `need` 是否已经出现，再把当前值存入字典。

## 回溯路径复制

问题：

把 `path` 直接加入结果，会导致结果中保存的是同一个列表引用。

修正：

加入结果时使用 `path[:]`，保存当前路径快照。

## 动态规划边界

问题：

动态规划题如果没有先处理空数组、长度为 1 等边界，很容易在访问下标时出错。

修正：

每个 DP 函数先写清楚最小输入规模，再写状态转移。
