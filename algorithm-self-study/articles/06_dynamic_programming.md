# 动态规划入门：状态、转移与边界

## 学习背景

动态规划是算法学习中比较难的一部分，因为它要求先定义状态，再找出状态之间的关系。它适合解决有重叠子问题和最优子结构的问题。

## 三个关键问题

学习动态规划时，我会按顺序问三个问题：

1. `dp[i]` 表示什么？
2. `dp[i]` 可以从哪些状态转移过来？
3. 初始状态和边界条件是什么？

只要这三个问题不清楚，就先不要急着写代码。

## 例子：爬楼梯

如果每次可以爬 1 或 2 阶，那么到达第 `i` 阶的方法数来自两个位置：第 `i - 1` 阶和第 `i - 2` 阶。

转移方程是：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

## 代码实践

`src/dynamic_programming.py` 中实现了：

- `climb_stairs`：爬楼梯方法数。
- `max_profit`：一次买卖股票的最大利润。
- `length_of_lis`：最长递增子序列长度。

## 复杂度分析

动态规划的复杂度取决于状态数量和每个状态的转移成本。爬楼梯可以优化到 `O(1)` 空间，因为每次只依赖前两个状态。

## 复盘

动态规划最重要的不是背公式，而是训练状态定义能力。以后遇到最值、方案数、可行性问题时，我会优先尝试从状态和转移角度分析。
