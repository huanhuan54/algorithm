# 图搜索：BFS 与 DFS

## 学习背景

树是一种没有环的特殊图。学习树遍历后，再看图搜索会更容易理解。图和树最大的区别是可能存在环，所以遍历时必须记录已经访问过的节点。

## DFS

深度优先搜索会沿着一条路径尽可能走到底，再回退到上一个分支。它适合判断连通性、寻找路径、处理需要递归展开的问题。

## BFS

广度优先搜索会一层一层扩展节点。它适合求无权图中的最短步数，因为第一次到达某个节点时，使用的边数就是最少的。

## 代码实践

`src/graph.py` 中实现了：

- `dfs_order`：返回从起点开始的 DFS 访问顺序。
- `bfs_order`：返回从起点开始的 BFS 访问顺序。
- `shortest_path_length`：求无权图中两个节点之间的最短边数。

图使用邻接表表示，例如：

```python
graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": ["D"],
    "D": [],
}
```

## 复盘

图搜索比树遍历多出的关键步骤是 `visited`。如果忘记记录访问状态，在有环图中就可能无限循环。
