# 算法学习路线与复杂度分析

## 学习背景

刚开始学习算法时，最容易陷入两个误区：第一是只背题解，第二是只关心代码能不能运行。算法学习更重要的是理解“为什么这样做更快”，而复杂度分析就是这件事的工具。

## 核心概念

时间复杂度描述输入规模变大时，程序运行时间增长的趋势。常见复杂度从低到高大致包括 `O(1)`、`O(log n)`、`O(n)`、`O(n log n)`、`O(n^2)`。空间复杂度描述额外内存的增长趋势。

例如，在数组中查找一个目标值，如果逐个扫描，需要 `O(n)` 时间；如果数组已经有序，使用二分查找可以把时间降到 `O(log n)`。

## 学习路线

我的自学路线分为四个阶段：

1. 基础阶段：数组、字符串、链表、栈、队列、哈希表。
2. 思维阶段：双指针、滑动窗口、二分查找、前缀和。
3. 搜索阶段：递归、回溯、树、图的遍历。
4. 进阶阶段：动态规划、贪心、并查集、最短路。

## 代码实践

本仓库的 `src/search.py` 中实现了线性查找和二分查找。二分查找的关键是维护闭区间 `[left, right]`，每轮比较中点值并缩小搜索范围。

```python
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        if nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

## 复盘

复杂度不是为了写出“看起来高级”的代码，而是为了在题目约束下选择合适方法。以后做题时，我会先看数据规模，再判断暴力解是否可行，最后再决定是否需要优化。

## 补充：二分查找不只用于找值

第二次复习二分查找时，我发现“找到目标值”只是最基础的用法。很多题真正需要的是找到第一个大于等于目标值的位置，也就是 `lower_bound`；或者找到第一个大于目标值的位置，也就是 `upper_bound`。

这类写法更适合处理插入位置、重复元素边界、答案二分等问题。为了减少闭区间边界带来的混乱，我在 `lower_bound` 和 `upper_bound` 中使用左闭右开区间 `[left, right)`。

对比下来：

- `binary_search`：回答目标值是否存在，存在则返回一个下标。
- `lower_bound`：回答目标值应该插入到哪里，保证左侧都小于目标值。
- `upper_bound`：回答目标值右边界在哪里，保证左侧都小于等于目标值。
